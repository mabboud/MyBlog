This is the framework, using which I implemented Incremental Search to any Swing Component. This framework consists of just a single Action class.

// @author Santhosh Kumar T - santhosh@in.fiorano.com 
13   public abstract class FindAction extends AbstractAction implements DocumentListener, KeyListener{ 
14       JPanel searchPanel = new JPanel(new FlowLayout(FlowLayout.LEFT, 0, 0)); 
15       protected JTextField searchField = new JTextField(); 
16       JPopupMenu popup = new JPopupMenu(); 
17    
18       public FindAction(){ 
19           super("Incremental Search"); //NOI18N 
20           searchPanel.setBackground(UIManager.getColor("ToolTip.background")); //NOI18N 
21           searchField.setOpaque(false); 
22           JLabel label = new JLabel("Search for:"); 
23           label.setFont(new Font("DialogInput", Font.BOLD, 12)); // for readability 
24           searchPanel.add(label); 
25           searchPanel.add(searchField); 
26           searchField.setBorder(BorderFactory.createEmptyBorder(0, 5, 0, 0)); 
27           searchPanel.setBorder(BorderFactory.createEmptyBorder(0, 5, 0, 5)); 
28           popup.setBorder(BorderFactory.createLineBorder(Color.black)); 
29           popup.add(searchPanel); 
30           searchField.setFont(new Font("DialogInput", Font.PLAIN, 12)); // for readability 
31    
32           // when the window containing the "comp" has registered Esc key 
33           // then on pressing Esc instead of search popup getting closed 
34           // the event is sent to the window. to overcome this we 
35           // register an action for Esc. 
36           searchField.registerKeyboardAction(new ActionListener(){ 
37               public void actionPerformed(ActionEvent e){ 
38                   popup.setVisible(false); 
39               } 
40           }, KeyStroke.getKeyStroke(KeyEvent.VK_ESCAPE, 0), JComponent.WHEN_FOCUSED); 
41       } 
42    
43       protected JComponent comp = null; 
44       protected boolean ignoreCase; 
45    
46       /*-------------------------------------------------[ ActionListener ]---------------------------------------------------*/ 
47    
48       public void actionPerformed(ActionEvent ae){ 
49           if(ae.getSource() == searchField) 
50               popup.setVisible(false); 
51           else{ 
52               comp = (JComponent)ae.getSource(); 
53               ignoreCase = (ae.getModifiers() & ActionEvent.SHIFT_MASK)!=0; 
54    
55               searchField.removeActionListener(this); 
56               searchField.removeKeyListener(this); 
57               searchField.getDocument().removeDocumentListener(this); 
58               initSearch(ae); 
59               searchField.addActionListener(this); 
60               searchField.addKeyListener(this); 
61               searchField.getDocument().addDocumentListener(this); 
62    
63               Rectangle rect = comp.getVisibleRect(); 
64               popup.show(comp, rect.x, rect.y - popup.getPreferredSize().height - 5); 
65               searchField.requestFocus(); 
66           } 
67       } 
68    
69       // can be overridden by subclasses to change initial search text etc. 
70       protected void initSearch(ActionEvent ae){ 
71           searchField.setText(""); //NOI18N 
72           searchField.setForeground(Color.black); 
73       } 
74    
75       private void changed(Position.Bias bias){ 
76           // note: popup.pack() doesn't work for first character insert 
77           popup.setVisible(false); 
78           popup.setVisible(true); 
79    
80           searchField.requestFocus(); 
81           searchField.setForeground(changed(comp, searchField.getText(), bias) ? Color.black : Color.red); 
82       } 
83    
84       // should search for given text and select item and 
85       // return true if search is successfull 
86       protected abstract boolean changed(JComponent comp, String text, Position.Bias bias); 
87    
88       /*-------------------------------------------------[ DocumentListener ]---------------------------------------------------*/ 
89    
90       public void insertUpdate(DocumentEvent e){ 
91           changed(null); 
92       } 
93    
94       public void removeUpdate(DocumentEvent e){ 
95           changed(null); 
96       } 
97    
98       public void changedUpdate(DocumentEvent e){} 
99    
100      /*-------------------------------------------------[ KeyListener ]---------------------------------------------------*/ 
101   
102      protected boolean shiftDown = false; 
103      protected boolean controlDown = false; 
104   
105      public void keyPressed(KeyEvent ke){ 
106          shiftDown = ke.isShiftDown(); 
107          controlDown = ke.isControlDown(); 
108   
109          switch(ke.getKeyCode()){ 
110              case KeyEvent.VK_UP: 
111                  changed(Position.Bias.Backward); 
112                  break; 
113              case KeyEvent.VK_DOWN: 
114                  changed(Position.Bias.Forward); 
115                  break; 
116          } 
117      } 
118   
119      public void keyTyped(KeyEvent e){} 
120      public void keyReleased(KeyEvent e){} 
121   
122      /*-------------------------------------------------[ Installation ]---------------------------------------------------*/ 
123   
124      public void install(JComponent comp){ 
125          comp.registerKeyboardAction(this, KeyStroke.getKeyStroke('I', KeyEvent.CTRL_MASK), JComponent.WHEN_FOCUSED); 
126          comp.registerKeyboardAction(this, KeyStroke.getKeyStroke('I', KeyEvent.CTRL_MASK|KeyEvent.SHIFT_MASK), JComponent.WHEN_FOCUSED); 
127      } 
128  }

Lines Description
13  I used Action Subclass for as the base. It allows you to use it very flexibly. for example changing keystrokes, enabling/disabling etc.
14-16 The search panel is actually a JPopupMenu, which contains a label with text "Search for:" and a JTextField
20-30 creates search popup. I used tooltip background, to distinguish it from celleditors. Font is changed to make it more readable.
36-40 Use escape to close the popup. It is taken care that Esc keystroke doesn't close the dialog by registering it as keyboardAction
124-127 This is the method used to install Incremental Search to a SwingComponent.
By default this action is registered for two KeyStrokes:
CRTL+I = case sensitive search
CTRL+SHIFT+I = case insensitive search

subclasses can override this to add more shortcuts. For example, for JTextComponent we may want to add CTRL+ALT+I - for incremental search by regular expressions.
 
48-67 adds various listeners and initializes the search - calls initSearch() method
Popup is shown 5 pixels above the component.
70-73 initializes the search. Subclasses may override this to do some extra work.
for example: for JTextComponent, when incremental search is invoked we might want the search text field to be pre-filled by the selected text.
75-82 this is the method that gets called as the user types in text in search text field or presses UP/DOWN arrow keys. Note: this is private method. It actually calls another abstract method changed(...) which needs to be implemented by subclasses.
The argument bias can be Forward or Backward or null. It tells the direction in which search to be performed. If it is null, the subclasses are free to choose either forward or backward (but mostly null is taken as forward)
86 This is the only abstract method that needs to be implemented by the subclasses. This should returns true, if an item is found on search. This return value is used to change the foreground color of search textfield to red if no item matches.
90-98 There are document listener methods which call chaged(...) on any change
105-120 Listens for up/down arrow keys in textfield and calls changed(....). it stores the current modifiers (control/shift pressed) in protected variables which can be used by subclasses to effect the type of selection

This class does all the important work of showing/hiding popup, initiate/perform search etc...
