Today I am giving a programming challenge to my blog readers on Drag-n-Drop.

The application contains 3 lists named personList, employeeList, ManagerList and one textEditor.

We have 3 class in our application:

Person extends Object;
Employee extends Person;
Manager extends Employee;

Constraints on Lists' data:

  • personList can hold only persons (i,e Object of type Person, Employee, Manager)
  • employeeList can hold only employees (i,e Object of type Employee, Manager)
  • managerList can hold only managers (i,e Object of type Manager)

Now the challenge is implementing dnd for all the lists and the textEditor.

You should allow the user to move list items from one to another, but should disallow the dnd which result a violation of above rule.

i,e,

user should be allowed to drop a Manager object to the personList, employeeList and managerList.
user should be allowed to drop a Employee object to either personList or employeeList
user should not be allowed to drop Person object only to personList.

Any Object can be dropped onto the editor and on drop it should insert that object name(i,e toString) in the textEditor.

I have solved this problem. So you can run it using webstart to clear the above said ideas.

You can download the sources for this challenge here. Note that the downloaded sources just contains the complete application without any dnd. The challenge is implementing dnd for this application.

Once again I am telling. solving the challenge is not important, the important thing is how effective and reusable you solution is. If you sincerely try this, You will learn lot of things about dnd.
